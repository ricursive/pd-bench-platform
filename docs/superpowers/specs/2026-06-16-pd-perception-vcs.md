# `pd` — GitHub for Physical Design (perception + versioning)

Status: P0–P2 implemented in [`pd/`](../../../pd), demoed on the ariane133 fixture. P3–P5 pending.

## Thesis
SWE-bench agents iterate well because **code is already a representation an LLM can
read**, and git/GitHub version + diff + review it. A DEF is machine geometry —
illegible. The missing layer for physical-design agents is therefore not a solver
or an agent harness (that's the agent's job to build), but an **LLM-legible,
version-controlled representation of physical state**: the "GitHub for physical
design." We expose the physics; the agent brings the engineering.

This directly tests the hypothesis that models like codex score low (3.37) because
the run is an **open loop** — they can't see the chip, get no incremental metric,
can't diff attempts — not because they can't reason.

## Principles
- **Text-first** (we evaluate LLMs, not VLMs): ASCII spatial maps + structured
  tables + textual diffs. Images stay for the human/VLM web layer only.
- **Read-only**: `pd` perceives + versions; it never mutates. The agent changes the
  design with its own EDA tools (TCL/OpenROAD); `pd` observes the resulting DEF and
  tracks it — git over an editor. No `place`/`move`/`route` verbs.
- **Hierarchical + zoomable** (LOD): `pd view` (chip) → `pd view region(c,r)` →
  `pd view <cell> --context`, like `ls` → `ls -la` → `cat`. The agent picks the
  granularity its decision needs, so perception stays inside the context budget.
- **Hierarchy mirrors the design**: `.pd/blocks/<module>/` per partition; the agent
  navigates blocks like a filesystem (`pd ls`, `pd view <block>`).
- **Parity**: views are generated by the verifier's own `def_checks`/`scoring`
  (vendored pd-bench), so perception and grading can't drift.

## What's built (`pd/`)
- `_bench.py` — bridge to the submodule's `def_checks`/`scoring` (parity).
- `model.py` — `Design`/`Inst`: hierarchy-aware, bbox-resolved instances + nets.
- `metrics.py` — instant `hpwl_proxy` (instance-center) + `gate_report` (the real gates).
- `views.py` — LOD text views: `chip`, `region(c,r)`, `cell(--context)`, `block`, `ls`, `status`.
- `diff.py` — physical+textual diff: moved insts + displacement, region density Δ, HPWL Δ, set changes.
- `store.py` — content-addressed read-only VCS: `commit/log/resolve/blob`; first commit = floorplan = gate reference.
- `cli.py` — `pd init|commit|log|ls|status|view|diff`. Perception + versioning verbs only.

## Before / after (trace)
**Before:** `head` a 619 KB DEF → guess TCL → run → wall of unlocalized violations → submit blind → `valid 0` / 3.37.
**After:** `pd view` (legible chip + gates + HPWL) → run own flow → `pd status` localizes
"`d_cache/c8500` 1.87µm from `sram82` (need ≥2.00)" → `pd view region(20,10)` / `pd view d_cache/c8500 --context` to drill → fix → `pd diff HEAD~1 HEAD` ("moved 8723 insts, HPWL −x%") → commit. The trace becomes a legible commit graph grounded in physical state.
